### Numeric Features
- overall_qual (int, 1-10): Overall material and finish quality. 1 = Very Poor, 10 = Very Excellent.
- gr_liv_area (int, 334-5095): Above grade living area in square feet.
- garage_area (int, 0-1488): Garage size in square feet. 0 = no garage.

- year_built (int, 1872-2010): Original construction year. If the user says a year after 2010, use 2010.
- full_bath (int, 0-4): Number of full bathrooms above grade.
- mas_vnr_area (int, 0-1290): Masonry veneer area in square feet. 0 = none.

### Quality Features (use exactly one of: NA, Po, Fa, TA, Gd, Ex)

- kitchen_qual: Kitchen quality. Same scale.
- fireplace_qu: Fireplace quality. NA = no fireplace. Same scale.

## Rules
1. Return null for any feature NOT mentioned — do NOT guess or fill defaults.
2. "NA" is a real value meaning the feature is absent (no basement, no fireplace). null means not mentioned.
3. Only return the JSON object with the 12 fields. No extra fields, no explanations.
4. Map qualitative words: "excellent/beautiful/top" → Ex, "good/nice" → Gd, "average/typical/decent/ok" → TA, "fair/mediocre" → Fa, "poor/bad" → Po.
5. If the user mentions bedrooms, that is not one of the 12 features — ignore it.
